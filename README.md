## 提交规范
业界主流的 Git 提交规范 Conventional Commits 中的提交类型包括 feat, fix, docs, refactor, chore 等，但我总感觉不太适合这个我的学习仓库，所以在豆包的基础上，我做了一些简单的提交类型，我在更新本仓库时会使用这些提交类型。

note：笔记，总结相关的内容更新<br>
code：project, lab 的代码功能增加或减少<br>
update：project, lab 的代码功能，结构优化<br>
add：project, lab 代码模板等从其它地方添加的文件<br>
docs：README的更新<br>
refactor：仓库文件结构的更新<br>
finish：标志着某门网课的某一个 project, lab 完成，某本教材的某一章等完成<br>
learn：标志着某门网课，某本教材等的学习完成
